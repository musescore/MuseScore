Development builds of MuseScore

The development builds allow you to test the very latest developments, including new features and bug fixes. The goal is to catch new bugs, crashes, and regressions before they reach the prerelease or stable versions. For more information, see Comparison of stable, beta, and nightly versions. You can report any regressions or bugs you find via Issues Tracker.

A development build can run side-by-side with other versions of MuseScore on the same computer. If you change preferences in the nightly build, it does not affect your preferences in the prerelease or stable versions of MuseScore. You can use the latest stable version of MuseScore from musescore.org.
